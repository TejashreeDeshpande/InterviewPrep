## How to use

Each assignment contains:
1. Problem statement & real-world scenario
2. Functional & non-functional requirements
3. Constraints & edge cases
4. Expected architecture diagram
5. Sample input/output
6. Suggested APIs/classes
7. Follow-up interview questions
8. Common mistakes candidates make
9. Optimization discussion
10. Full production-quality Kotlin solution
11. Alternative approaches & tradeoffs
12. Unit test examples

## Difficulty guide

- 🟢 **Easy** — Kotlin fundamentals, coroutine basics
- 🟡 **Medium** — Jetpack Compose, StateFlow/SharedFlow, MVI
- 🔴 **Senior** — Architecture, offline-first, performance, advanced concurrency
- 🟣 **Staff** — System design, distributed mobile systems, cross-cutting concerns
